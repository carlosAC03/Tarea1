Planificación y Analisis de Requisitos

La tienda va a ser una distribuidora de funkos con un catalogo extendido con diferentes unidades cada una con un rareza y precio diferente

Requisitos Fundamentales

Catálogo.

Se organizaran los productos por tipos:

  • Producto
  • Precio
  • Exclusividad

Inicio de sesión.

Podran registrase con su correo y mantener la sesión iniciada si lo desean

Historial.

Se guardaran los productos ya comprados en el historial para ver sus caracteristicas

Diferentes tipos de pago.

Podran pagar con tarjeta o paypal

Requisitos no Funcionales

Usabilidad

Tendra una interfaz intuitiva , resaltando cada apartado importante

Compativilidad

Tendra soporte para todas la plataformas tanto ios, android, windows...

Escalavilidad

Manejaremos la cantidad de productos que nos llegan de cada sitio

Diseño de la arquitectura

  • Utilizaremos React para una interfaz ágil y responsiva, adaptada a diferentes.
  • Backend con Node.js
  • Base de datos MongoDB
  • Integración de la pasarela de pago Stripe para tarjetas

    • Implementación

    1. Configurar la API REST para gestionar el catálogo de productos, el carrito y los pagos.
    2. Crear interfaces atractivas con React, incluyendo un sistema de recomendación de productos.
    3. Integrar las opciones de pago y configurar los cupones de descuento en tiempo real.

    Pruebas

    Realizar pruebas de carga con herramientas como JMeter para ver cómo responde la plataforma bajo la demanda de múltiples usuarios.

    Testear todos los flujos de compra y la funcionalidad del carrito con diferentes dispositivos y navegadores.

    Despliegue y Mantenimiento

    Desplegar la aplicación en AWS para asegurar escalabilidad, con balanceadores de carga y respaldo automatizado.

    Monitorear el rendimiento de la aplicación usando CloudWatch, y establecer ciclos de actualización periódicos para mejoras y nuevos lanzamientos de productos.
